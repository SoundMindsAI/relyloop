### Conventions (project-specific)

```
- The generator is a single Python file at website/scripts/build_guides.py, STDLIB-ONLY
  (re, pathlib, json, subprocess, os, sys, argparse, urllib.parse). NO PyYAML, NO third-party imports.
- Module shape mirrors ui/scripts/copy-docs.mjs: importing the module does NOT trigger generation;
  a bottom-of-file `if __name__ == "__main__":` entrypoint gates the run. Public functions are
  importable by the pytest at backend/tests/unit/scripts/test_build_guides.py.
- The generator exposes named, individually-testable functions: rewrite_link(), build_video_block(),
  emit_deck_page(), emit_index(), render_nav_fragment(), splice_nav_fragment(), prune_dir(),
  transcode_webm_to_mp4(), main(). Each is unit-testable against tmp fixtures with NO filesystem
  dependency on the real repo (paths injected as args).
- Generated .md files begin with the literal line:
  <!-- GENERATED by website/scripts/build_guides.py from <SOURCE> — DO NOT EDIT. -->
- Repo-root discovery: the generator resolves repo root via pathlib relative to __file__
  (website/scripts/build_guides.py → parents[2]), NOT via `git rev-parse` (keeps it hermetic).
- Self-test bash scripts mirror scripts/ci/test_verify_copy_docs_fresh.sh exactly — git init in a
  tmp dir, no operation touches the operator's primary checkout, COPY_DOCS_FRESH_REPO_ROOT-style
  env override for the repo root.
- Python tests live at backend/tests/unit/scripts/test_build_guides.py (the established home for
  repo-script unit tests — see test_run_tests_in_worktree.py, test_seed_es_retry.py there).
```

```python
GITHUB_BLOB = "https://github.com/SoundMindsAI/relyloop/blob/main/"
GITHUB_TREE = "https://github.com/SoundMindsAI/relyloop/tree/main/"
INLINE_LINK_RE = re.compile(r"(?<!\!)\[(?P<text>[^\]]*)\]\((?P<url>[^)]+)\)")  # inline links, NOT images
PRESENTER_COMMENT_RE = re.compile(r"<!--\s*presenter:.*?-->", re.DOTALL)

class LinkRewriteError(ValueError):  # carries the offending target only
    def __init__(self, target: str, reason: str): ...

def rewrite_link(url: str, repo_root: Path) -> str: ...
    # urlsplit(url) -> (scheme, netloc, path, query, fragment)
    # rule 1: scheme != "" -> passthrough  (covers https, http, mailto, tel, data)
    # rule 2: scheme=="" and netloc=="" and path=="" and fragment!="" -> passthrough (bare #anchor)
    # rule 3: path matches ^(?:\./)?<one of LONG_FORM_GUIDES>$ -> strip ./, reattach query+frag
    # rule 4: path starts with ../ -> normalize vs docs/08_guides/, reject repo-escape, assert exists,
    #         emit blob (file) or tree (dir) URL, reattach query+frag
    # else -> raise LinkRewriteError(target, "unmapped"/"unresolved"/"repo-escape")
    # NB (C1-A1 fix): rewrite_link does NOT know file:line. It raises LinkRewriteError with the
    # offending target + reason; port_long_form_guide() catches it, knows the source file + the
    # line number of the match, and re-raises SystemExit(1) with the full "file:line ... target" msg.

def detect_unsupported_relative_links(text: str, source_basename: str) -> None: ...
    # scans for image links / reference-style / <a href> / autolinks whose target is relative
    # (urlsplit scheme=="" and netloc=="") and starting with ./, ../, or bare-basename; raises if found

def port_long_form_guide(src: Path, repo_root: Path) -> str: ...
    # strips presenter comments, rewrites every inline link via rewrite_link (catching
    # LinkRewriteError and re-raising SystemExit(1) with f"ERROR: {src.relative_to(repo_root)}:{line_no} ...")
    # using the match's line number, prepends GENERATED header.
    # (C2-B3) A missing src for a hard-coded LONG_FORM_GUIDES entry raises SystemExit(1)
    # "ERROR: long-form guide source missing: docs/08_guides/<basename>" — these 4 are a fixed,
    # always-present set (unlike dynamically-discovered decks), so a missing one is a real error.
    # This also prevents a stale generated in-depth/<basename>.md surviving when its source is
    # deleted but prune's expected set still hard-codes it (FR-6 gate scope excludes docs/08_guides/,
    # so fail-loud at the source is the only signal).
```
